1、何为守护进程？
	（1）daemon:守护进程、 后台程序简称是d，（一般带有d的都是守护进程）
	（2）长期运行： 从系统启动时开始运行，到系统关闭时结束(运行周期)
	（3）与控制台脱离（终端）：守护进程启动后，脱离控制台，不受终端控制，背后的问题是会话
	（4）服务器：是一个应用程序，使用时直接调用

定义： 	是一个在后台运行，并且不受终端控制的服务程序！

3、编写守护进程

	实现守护进程的步骤：

（1）创建父子进程，父进程退出  （孤儿进程）

（2）在子进程中设置新的会话   （setsid）

（5）关闭所有文件描述符

【3】线程

线程是什么？
	线程是一个轻量级的进程，内核也会调度线程，线程在进程当中创建。
	
	  具有执行力、独立的代码块。
进程是什么？进程 = 线程 + 资源。
	根据进程内线程的数量，进程可分为单线程进程和多线程进程。

进程与线程的关系：
	进程拥有整个地址空间，在这个进程中装有线程所需的资源，所以地址空间就相当于资源容器就像与岗位与提供了水。因此进程和线程的关系是进程是资源容器，线城是资源拥有者。

进程和线程的区别：
	线程没有自己独享的资源，因此没有自己的地址空间，他要依附在进程的地址空间中借助进程的资源运行。
	
	线程间如何实现通信？   全局变量、数组


	线程标识符：用来唯一的表示一个线程，通过调用线程库来实现对线程的访问操作，
	线程是用函数实现的
	1、创建       --》  pthread_create
	2、回收线程   --》  pthread_join
	3、结束线程   --》  pthread_exit
	4、线程销毁   --》  pthread_destroy

	编译程序时一定要加上 -pthread ，因为他是第三方提供的

	相关函数：
	int pthread_create(pthread_t *thread, const pthread_attr_t *attr,
                          void *(*start_routine) (void *), void *arg);

	功能：创建线程
	参数：
		thread   线程对象
		attr     线程属性默认为NULL，栈的空间大小是8M
		start_routine  函数指针，用来指向函数名
		arg      参数，作为函数指针的参数
		返回值：
		成功：  0
		失败：  错误码

void pthread_exit(void *arg)
功能：  线程结束函数
参数：  传递线程退出时的状态值

pthread_join（pthread_t pthread， void**retval）
功能：线程回收函数（阻塞等待，直到等到线程退出了，会立即释放）
参数： 
	pthread  线程对象
	retval   线程退出时的状态值


共享资源：
	可以被多个进程或者线程访问的资源（线程中的全局变量）

临界资源：
	只允许一个进程或者线程的共享资源
临界区：
	访问共享资源的那段代码

1、信号量：

	物理定义：信号量代表一类资源，资源的值表示信号量的数量
初始化信号量，也就是代表资源的数量是多少个
	目的：

	P（申请资源）
		申请两种情况：
		如果有资源，会继续运行相应代码，数量减1

		如果没有资源，会阻塞等待

	V（释放资源）
		如果没有任务在申请资源，让资源的数量1，
		如果有任务在申请资源资源的数量会减1




2、互斥锁

		临界资源：
		一次只允许一个被进程或者线程能够访问的共享资源
		举例： 全局变量  全局数组

		临界区：
			访问临界资源的代码

		互斥锁机制： 
			只允许一个线程或者进程申请锁，其他线程或者进程申请同一把锁，会阻塞，直到获得锁为止，阻塞线程才会释放，继续往下运行
